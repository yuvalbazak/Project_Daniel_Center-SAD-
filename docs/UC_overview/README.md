# Use Case Overview

This directory contains the use case diagrams, selected use case specifications, and supporting documentation for the **My Daniel Information System** project.

## Purpose

The documents in this folder describe how different users interact with the system and how the proposed solution supports the operational needs of Daniel Rowing Center.

The use case model serves as a bridge between the organizational analysis phase and the system requirements phase.

## Main Actors

| Actor | Description |
|--------|--------|
| Center Manager | Oversees organizational activities and reporting |
| Accounting Manager | Manages registrations, memberships, payments, and payroll |
| Coordinator | Schedules activities and manages operational resources |
| Instructor | Conducts activities and reports attendance |
| Customer | Registers for activities and participates in programs |
| Maintenance Staff | Handles boat and equipment maintenance |
| System Time | Triggers automated system processes |

1. Review the use case diagrams to understand system functionality.

3. Reference the MSS document for primary success scenarios.
4. Use these documents together with the requirements document when analyzing system behavior.
