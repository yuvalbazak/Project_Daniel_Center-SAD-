# Organization Analysis

This directory contains the organizational analysis documents for the **Daniel Rowing Center Information System (My Daniel)** Software Analysis and Design (SAD) project.

## Purpose

The documents in this folder support the **organizational analysis phase** of the Software Analysis and Design curriculum, focusing on understanding the organization's structure, stakeholders, business processes, operational challenges, and opportunities for system improvement.

The analysis serves as the foundation for the requirements engineering, use case modeling, and system design phases of the project.

## Contents

Documents in this directory provide:

- **Organizational context** — Structure, activities, departments, and information systems currently used by Daniel Rowing Center
- **Stakeholder analysis** — Interviews and observations collected from key organizational stakeholders
- **Business problem analysis** — Identification of operational challenges, inefficiencies, and opportunities for improvement
- **Business process documentation** — Description and modeling of the organization's core operational processes
- **Process modeling artifacts** — BPMN diagrams representing current-state business workflows

## Directory Structure

| File | Description |
|--------|--------|
| `01-organization.md` | Organization profile, structure, activities, and existing information systems |
| `02-interviews.md` | Stakeholder interviews and field observations |
| `03-problems.md` | Business problems analysis and improvement opportunities |
| `04-business-processes.md` | Core business process descriptions and workflow analysis |
| `bpmn-bp01-*` | Instructor scheduling, attendance reporting, and payroll synchronization BPMN |
| `bpmn-bp02-*` | Customer registration and membership management BPMN |
| `bpmn-bp03-*` | Boat and equipment maintenance management BPMN |

## Related Documents

- **`docs/00-requirements.md`** — Functional and non-functional requirements derived from the organizational analysis
- **`docs/00e-use-cases.md`** — Use case model and actor interactions
- **System Design Documents** — Future design artifacts based on the findings documented in this folder

## How to Use This Folder

1. Start here when learning about Daniel Rowing Center and its operational environment.
2. Review stakeholder interviews to understand business needs and pain points.
3. Examine the identified business problems before reviewing requirements.
4. Use the BPMN diagrams to understand current operational workflows.
5. Reference these documents when validating requirements traceability and system design decisions.
