# Organization Analysis

This directory contains organizational analysis documents for the Daniel Center order management system SAD project.

## Purpose

The documents in this folder support the **analysis phase** of the Software Analysis and Design curriculum, specifically focusing on organizational context, domain understanding, and stakeholder requirements.

## Contents

Documents in this directory provide:

- **Organizational context** — How the Daniel Center operates, key business processes, and stakeholder roles
- **Domain understanding** — The order management domain, workflows, and business rules specific to the Daniel Center
- **Background for requirements** — Foundation that led to the user stories and NFRs in `docs/00-requirements.md`

## Related Documents

- **`docs/00-requirements.md`** — User stories (65) and non-functional requirements (28) derived from this organizational analysis
- **`docs/00e-use-cases.md`** — Use case specifications with implementation notes
- **`CLAUDE.md`** — Project overview and architecture conventions
- **`PATTERNS.md`** — Shared SAD conventions across all student projects

## How to Use This Folder

1. **Start here** when understanding the Daniel Center's business context
2. **Reference** these documents when validating requirements traceability
3. **Review** organizational roles and workflows before designing the system architecture
