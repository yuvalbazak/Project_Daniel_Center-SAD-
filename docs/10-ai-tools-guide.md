# שימוש בכלי AI בפרויקט

מדריך זה מסביר כיצד להשתמש בכלי AI (כמו Claude, ChatGPT, Copilot) באופן **מועיל ומתודי** בפרויקט.

## 1. הכלל המרכזי

> **AI הוא כלי עזר — לא מפתח.**
> אתם אחראים להבין כל שורת קוד שנכנסת לפרויקט.
> אם אתם לא יכולים להסביר מה הקוד עושה — אל תשתמשו בו.

## 2. הדרך הנכונה: עבודה מתודית

### ❌ הדרך הלא נכונה — "Vibe Coding"
```
"תבנה לי מערכת לניהול חנות עם עובדים, מוצרים, הזמנות,
לקוחות, ודוחות. תשתמש ב-WinForms ו-SQL Server."
```

**למה זה לא עובד:**
- מקבלים קוד ענק שאי אפשר להבין
- המבנה לא תואם את מה שלמדתם
- אי אפשר לתקן באגים כי לא מבינים את הקוד
- אי אפשר להוסיף פיצ'רים כי לא מבינים את הארכיטקטורה

### ✅ הדרך הנכונה — שלב אחרי שלב

תנו ל-AI את **מסמך הדרישות שלכם** (User Stories + Class Diagram) ועבדו **ישות אחת בכל פעם**, בסדר הפיתוח שלמדתם.

```
└──────────────────────────────────────────────────┘
```

## 4. איך לתת הקשר נכון ל-AI

### בהתחלה — תנו את המסמכים שלכם

```
הנה מסמך הדרישות שלי לפרויקט:
- User Stories: [הדביקו את ה-User Stories]
- Class Diagram: [הדביקו תמונה או תיאור]
- בסיס הנתונים: SQL Server
- טכנולוגיה: C# WinForms, .NET 8

אני רוצה לעבוד על ישות אחת בכל פעם, בסדר הבא:
1. טבלה + Stored Procedures
2. מחלקת Entity ב-C# (לפי הדפוס של Worker.cs)
3. הוספה ל-Program.cs
4. פאנל תצוגה
```

### לכל ישות — תנו הקשר ספציפי

```
אני עובד על ישות Customer.
הנה ה-Class Diagram:
- customerId: string (PK)
- customerName: string
- phone: string
- email: string
- קשר: Customer 1--* Order

הנה דוגמה למחלקה קיימת בפרויקט שלי (Worker.cs):
[הדביקו את Worker.cs]

כתוב לי את Customer.cs באותו דפוס בדיוק.
```

> **טיפ חשוב:** תמיד תנו ל-AI דוגמה מהקוד הקיים שלכם.
> ככה הוא ייצר קוד שתואם את הסגנון והמבנה של הפרויקט.

## 5. מה לבקש ומה לא

### ✅ בקשות טובות (דבר אחד, ספציפי):
- "כתוב לי את טבלת Customers ב-SQL Server עם Primary Key ו-NVARCHAR לשמות"
- "כתוב Stored Procedure להוספת לקוח חדש"
- "כתוב את מחלקת Customer.cs באותו דפוס כמו Worker.cs"
- "כתוב את initCustomers() לפי הדפוס של initWorkers()"
- "כתוב פאנל CustomerPanel שמציג את כל הלקוחות ב-DataGridView"
- "יש לי באג: כשאני מוסיף לקוח מקבל שגיאה X. הנה הקוד: [...]"

### ❌ בקשות גרועות (גנריות, גדולות):
- "תבנה לי את כל המערכת"
- "תוסיף לי את כל הישויות בבת אחת"
- "תבנה לי ממשק משתמש יפה"
- "תתקן לי את הפרויקט" (בלי לציין מה השגיאה)

## 6. בדיקת קוד שקיבלתם מ-AI

לפני שמכניסים קוד לפרויקט, בדקו:

### צ'קליסט לבדיקת קוד מ-AI

- [ ] **מבינים כל שורה** — אם יש שורה שלא מבינים, שאלו את ה-AI מה היא עושה
- [ ] **תואם את הדפוס** — האם הקוד נראה כמו שאר הקבצים בפרויקט?
  - שמות מתודות ב-camelCase? (`createCustomer`, לא `Create_Customer`)
  - משתמש ב-`SQL_CON`? (לא יצר חיבור DB חדש)
  - משתמש ב-`mainForm.showPanel()`? (לא `Show()`/`Hide()`)
  - משתמש ב-`is_new` בבנאי?
- [ ] **בודק null** — האם יש בדיקת null אחרי כל `seek`?
- [ ] **טיפוסים נכונים** — האם טיפוסי ה-SQL תואמים את ה-C#?
- [ ] **מתקמפל** — הריצו Build לפני שממשיכים
- [ ] **עובד** — הריצו את התוכנית ובדקו שהפיצ'ר עובד

## 7. דברים ש-AI לא יודע

AI לא מכיר את הפרויקט שלכם אלא אם תספרו לו. הוא עלול:

| בעיה | דוגמה | מה לעשות |
|------|-------|---------|
| להמציא שמות טבלאות | `CREATE TABLE Customer` במקום `Customers` | תנו לו את שמות הטבלאות האמיתיים |
| להשתמש בדפוס אחר | ליצור `using` statement במקום `SQL_CON` | תנו לו דוגמה מהקוד שלכם |
| לשכוח את ה-Connection String | להשתמש ב-connection string אחר | הזכירו שיש `SQL_CON.cs` |
| ליצור Form במקום Panel | `public class X : Form` | הזכירו שאתם משתמשים ב-UserControl |
| לא לדעת על initLists | לא להוסיף את ה-init ל-Program.cs | בדקו בעצמכם ש-initLists מעודכן |

## 8. שימוש ב-AI לדיבוג

AI מצוין לפתרון באגים — אבל תנו לו הקשר מלא:

### ✅ בקשת דיבוג טובה:
```
מקבל את השגיאה הבאה:
"System.FormatException: The input string '' was not in a correct format"

זה קורה כשאני לוחץ על כפתור "הוסף" בפאנל CreateCustomerPanel.

הנה הקוד של הכפתור:
[הדביקו את המתודה]

הנה ה-Stored Procedure:
[הדביקו את ה-SP]
```

### ❌ בקשת דיבוג גרועה:
```
"הפרויקט שלי לא עובד, תתקן"
```

## 9. סיכום — 5 כללים לשימוש ב-AI

1. **תנו הקשר** — Class Diagram, דוגמה מהקוד, שם הישות
2. **עבדו בשלבים** — ישות אחת בכל פעם, בסדר הפיתוח
3. **תנו דוגמה** — "כתוב כמו Worker.cs" עובד טוב יותר מ-"כתוב מחלקה"
4. **בדקו הכל** — אל תעתיקו בלי להבין
5. **אתם האחראים** — אם לא מבינים את הקוד, אל תשתמשו בו
